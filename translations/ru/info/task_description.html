Ты когда нибудь слышал о таком языке разметки как YAML. Он представляет из себя дружественный формат серилизации данных. На столько дружественный, что его стало удобно читать и людям и программам. Поиграться с самими стандартом можно по этой ссылке .

YAML - это текст, а вам надо преобразовать его в объект. Но весь стандарт YAML я вас реализовывать не прошу, будем его реализовывать шаг за шагом.

Первый шаг - это преобразование ключ-значение. Ключем может быть любая строка, из латинских букв и цифр. Значением может быть однострочная строка (которая состоит из пробелов, лалинских букв и цифр) или число (int)

Покажу несколько примеров:

name: Alex
age: 12

Преобразуется в объект.

{ 
  "name": "Alex",
  "age": 12
}

Обратите внимание, что число автоматически получило тип int

Еще один пример показывает, что строка может содержать пробелы

name: Alex Fox
age: 12

class: 12b

Будет преобразован в следующий объект.

{
  "age": 12, 
  "name": "Alex Fox", 
  "class": "12b"
}

Обратите внимание на несколько вещей. Между строкой "age" и строкой "class" есть пустая строка, которая не мешает парсингу. Класс начинается с цифр, но имеет буквы, а значит не может быть преобразован в цифры, поэтому его тип остался строка (str)

Input: Строка в формате

Output: Int.
